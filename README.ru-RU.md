# obs-censor

[English](README.md) | Русский

`obs-censor` - это плагин-фильтр для OBS Studio, который автоматически скрывает или показывает источник, к которому он прикреплен, в зависимости от времени воспроизведения в MPC-BE.

> Примечание: этот плагин был создан с помощью ИИ.

Плагин работает как **фильтр источника**:

- фильтр добавляется прямо к источнику, которым нужно управлять;
- он опрашивает `variables.html` у MPC-BE;
- он сверяет текущее время воспроизведения с заданными таймингами;
- он **показывает** источник внутри подходящих интервалов и **скрывает** его вне этих интервалов.

## Возможности

- фильтр источника OBS для управления видимостью по таймингам
- синхронизация с воспроизведением MPC-BE через HTTP (`variables.html`)
- несколько диапазонов времени в формате `HH:MM:SS-HH:MM:SS`
- автоматическое исправление перепутанных границ, например `01:28:19-01:26:21`
- поддержка сдвига таймингов в формате `+/-HH:MM:SS`
- локализация на английский и русский (`en-US`, `ru-RU`)
- поддержка Windows через WinINet и Linux через libcurl

## Как это работает

Фильтр запускает фоновый поток, который периодически запрашивает страницу состояния MPC-BE.

Позиция воспроизведения считывается из одного из полей:

- `<p id="positionstring">HH:MM:SS</p>`, или
- `<p id="position">milliseconds</p>`

Дальше фильтр:

1. разбирает заданные интервалы времени;
2. применяет необязательный сдвиг таймингов;
3. проверяет, попадает ли скорректированное время в любой из интервалов;
4. рендерит источник только тогда, когда время совпадает.

### Логика видимости

- **Внутри** заданного интервала -> источник видим
- **Вне** всех интервалов -> источник скрыт
- **Нет URL или нет таймингов** -> скрытие отключается, источник остается видимым

> Примечание: сейчас плагин управляет только **видимостью видео**. Аудио он не глушит.

## Использование в OBS

1. В MPC-BE включите веб-интерфейс и убедитесь, что `variables.html` доступен.
   - URL по умолчанию: `http://localhost:13579/variables.html`
2. В OBS Studio откройте источник, которым хотите управлять.
3. Добавьте к нему фильтр **MPC-BE Censor**.
4. Настройте параметры фильтра.

## Настройки фильтра

### MPC-BE URL

HTTP-адрес, откуда берется позиция воспроизведения.

По умолчанию:

```text
http://localhost:13579/variables.html
```

### Интервал опроса (мс)

Как часто фильтр проверяет состояние MPC-BE.

- Минимум: `500`
- Максимум: `5000`
- По умолчанию: `1000`

### Сдвиг таймингов

Сдвигает все тайминги без редактирования самого списка.

Допустимые форматы:

- `+00:00:05`
- `-00:00:05`
- `00:01:30`
- также поддерживаются просто секунды, например `15` или `-15`

Поведение:

- `+` задерживает срабатывание
- `-` ускоряет срабатывание

Примеры:

- `+00:00:05` -> состояние источника меняется на 5 секунд позже
- `-00:00:05` -> состояние источника меняется на 5 секунд раньше

### Тайминги

Укажите один или несколько интервалов в таком формате:

```text
00:03:45-00:03:49
00:10:00-00:10:12
01:28:19-01:26:21
```

Фильтр ищет диапазоны времени в тексте, поэтому комментарии после интервала допустимы:

```text
00:03:45-00:03:49 - короткое скрытие
00:10:00-00:10:12 - вторая сцена
```

Если диапазон введен в обратном порядке, он исправляется автоматически.

## Сборка

Команды сборки для Windows:

```powershell
cmake --preset windows-x64
cmake --build --preset windows-x64 --config RelWithDebInfo
```

Основной выходной файл:

```text
build_x64/RelWithDebInfo/obs-censor.dll
```

Структура для локального запуска после сборки находится здесь:

```text
build_x64/rundir/RelWithDebInfo/
```

## Установка

Если устанавливаете плагин вручную, **не копируйте только DLL**.

Нужны и файлы данных плагина, особенно файлы локализации:

```text
obs-censor/locale/en-US.ini
obs-censor/locale/ru-RU.ini
```

Если файлов локализации нет, OBS может показывать сырые ключи вроде:

```text
MpcBeCensorFilter.Name
MpcBeCensorFilter.Timings
```

Для локального тестирования используйте файлы из:

```text
build_x64/rundir/RelWithDebInfo/
```
